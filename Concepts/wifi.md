## WPS 취약점
- PIN 인증이 완료될 경우 실제 비밀번호를 자동으로 전달하도록 설계되어 있음.
- WPS는 8자리 PIN으로 라우터에 쉽게 연결하는 편의 기능인데, 설계상 치명적인 결함이 존재한다.
- 인증 과정에서 앞 4자리를 먼저 검증하는데, 틀릴 경우 명시적으로 응답을 보낸다.
- 맞을 경우 다음 단계로 진행하는데 이후 3자리만 검증하면 마지막은 체크섬이라서 총 7자리만 검증하면 비밀번호 탈취가 가능하다.
- 이론상 10^8 = 1억가지지만 실제 11,000번만에 크랙 가능

### 관리모드 / 모니터모드
- 관리모드(기본값) : 자신에게 오는 패킷만 수신
- 모니터모드 : 주변 모든 무선 패킷 수신
- 모니터 모드 활성화시 root 권한 필요.
```bash
# 무선 인터넷 정보 조회
iw dev
```

### reaver
- WPS 취약점 악용하는 도구.
- WPS의 설계상 문제로 라우터가 스스로 비밀번호를 알려주게 만든다.
```bash
reaver -i <monitor mode > -b <target BSSID (AP의 MAC 주소)> -v
```
